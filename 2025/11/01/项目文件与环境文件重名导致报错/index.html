学习安装强化学习库 Gymnasium 时，用于测试前置库 box2d-py 的项目文件因为与环境文件重名导致报错。

在给 Python 环境安装强化学习库 Gymnasium 前，需要安装一个前置库 box2d-py，它可以支持 LunarLander-v3 等需要 Box2D 环境支持的情景。运行情景的代码示例如下：

在这里我创建 Box2D.py 文件运行进行验证，遇到如下报错：

报错显示我并没有安装 Box2D 。但通过查询 pip list，可以看到是的确安装了这个库的：

由于这个库本身就不太好安装，因此在网上能搜到不少的报错解决方案，但将所有的方案都尝试一遍后仍然无果。最后去查询源文件，发现在该库中已经存在一个名为 Box2D.py 的文件了：

也就是说，文件名冲突导致环境导入了错误的模块。由于 Python 解释器会优先在当前目录而非第三方库路径中查找，我的 Box2D.py 测试文件被误当作 Box2D 库模块加载，阻断了正确模块的导入流程。将测试文件改名后，得到了成功的运行结果。这是一个我以往没有遇到过的导致报错的原因，因此在此记录。
